Hoisting Explained Simply

Posted on Jan 28, 2026 · 4 min read

Hoisting is JavaScript's behavior where variable and function declarations are moved to the top of their scope before code execution. This often confuses beginners.

Example 1: Variable Hoisting

console.log(x); // undefined
var x = 5;

Even though x is declared after the console.log, JavaScript hoists the declaration to the top. But only declarations are hoisted, not assignments.

Example 2: Function Hoisting

sayHello();

function sayHello() {
  console.log("Hello!");
}

Function declarations are fully hoisted, so you can call them before they appear in the code.
